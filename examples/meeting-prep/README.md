# Meeting Prep Briefing

Researches a person and their company before a call. Generates a one-page briefing with talking points.

## Setup

```bash
export ANTHROPIC_API_KEY=your-key
uv run briefing.py "Sarah Chen" "Sequoia Capital"
```

No `uv`? Use `pip install -r requirements.txt && python briefing.py "Sarah Chen" "Sequoia Capital"` instead.

That's it. Briefing prints to stdout and saves to `briefings/`.

## What You Get

```
# Briefing: Sarah Chen — Sequoia Capital

## Person Overview
Role, background, investment focus, notable public takes.

## Fund/Company Overview
Stage focus, recent deals, fund size.

## Recent News
3-5 bullet points from the last 6 months.

## Talking Points
3 specific conversation starters grounded in the research.
```

## How It Was Built

Generated by running `/sprint` in one conversation: discovery → hypotheses → spec → working code.

See `hypotheses.md` for the assumptions it was built to test, `product_requirements.md` for the spec, and `decision_log.md` for the full reasoning trail.

## Architecture

- **Surface:** Single API call with server-side web search
- **Model:** Sonnet 4.6 (fast, cheap — upgrade to Opus for better talking points)
- **Tools:** `web_search_20260209` (Anthropic server-side, no extra API keys)
- **Output:** Markdown to stdout + saved to `briefings/`
