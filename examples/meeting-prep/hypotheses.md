# Hypotheses

## H1: Founders spend 30+ minutes preparing for investor calls and find it painful
- **Risk:** If prep isn't actually painful, no one will use this — it's a nice-to-have, not a need.
- **Test:** 🗣️ conversation
- **How:** Ask 5 YC founders: "Walk me through the last time you prepped for an investor call. How long did it take? What was frustrating?" Look for specific pain, not polite agreement.
- **Status:** untested

## H2: Web search can surface relevant, recent intel about a person and their fund
- **Risk:** If the AI can't find useful information (e.g., the person has low online presence, or results are stale/wrong), the output is useless.
- **Test:** 🛠️ prototype
- **How:** Build the tool, run it on 10 real investor names. Check: does it find recent deals? Correct fund info? Or is it hallucinating?
- **Status:** untested

## H3: A one-page briefing format is more useful than a raw info dump
- **Risk:** If founders just want raw links/data, the structured briefing adds overhead without value.
- **Test:** 🗣️ conversation
- **How:** Show 3 founders a sample briefing vs. raw search results. Which do they prefer? Do they actually read the sections or skip to one part?
- **Status:** untested

## H4: Founders would use this before every call, not just important ones
- **Risk:** If it's only useful for "big" meetings, usage is too infrequent to build a habit.
- **Test:** 🗣️ conversation
- **How:** Ask: "Would you use this before a call with a friendly angel? Or only before a Sequoia partner meeting?" Listen for where they draw the line.
- **Status:** untested

## H5: The tool can run fast enough to be useful (under 60 seconds)
- **Risk:** If it takes 5 minutes, founders will just google it themselves. Speed is the whole point.
- **Test:** 🛠️ prototype
- **How:** Time the prototype end-to-end on 10 runs. Target: briefing delivered in under 60 seconds.
- **Status:** untested
